版本
为了提高透明度和洞察我们的发布周期，并努力保持向后兼容性，引导将保持在尽可能多的语义版本控制指引。

发布将被编号与跟进格式：

为<major>。<minor>。<patch>

并构建以下准则：

打破向后兼容性颠簸的主要
新产品没有破坏向后兼容性颠簸，未成年人
Bug修复和杂项的改变，撞击补丁
对于更多信息SemVer，请访问http://semver.org/。
